# 2026-04-19 工作日志

## 修复 connectivity_plus CPU 占用过高（第 N 轮迭代）

**问题**：FlClash 容器启动后 CPU 占用 90%+，根因是 connectivity_plus 无限重试

### 尝试过的方案及失败原因

| 方案 | 结果 | 原因 |
|------|------|------|
| 挂载宿主机 /run/dbus/system_bus_socket | ❌ CPU 仍高 | 群晖 DSM 不运行 NetworkManager，D-Bus 上无 NM 服务 |
| startapp.sh 启动 dbus-daemon + NM | ❌ Permission denied | app 服务以非 root 运行，无法操作 /var/run/dbus |
| cont-init.d 脚本启动 NM | ❌ SocketException 仍在 | cont-init 在 supervisor 之前执行，此时 dbus-daemon 还没跑，NM 等不到 socket |

### 最终方案：supervisor 服务

将 NM 作为 `/etc/services.d/networkmanager/run` 添加到 jlesage baseimage 的 supervisor：
- NM 服务与 dbus 服务并行启动（都在 supervisor 阶段）
- NM 的 run 脚本等待 `/var/run/dbus/system_bus_socket` 就绪后再启动 NM
- 这样 NM 能正确连上 D-Bus 并注册 `org.freedesktop.NetworkManager` 服务
- connectivity_plus 能查询到 NM → 不再重试 → CPU 正常

### jlesage/baseimage-gui 启动时序

```
cont-secrets → cont-env → cont-init.d (root, supervisor 未启动)
    → supervisor loading services → supervisor starting services
    → dbus, xvnc, nginx, openbox, app, [networkmanager] 等
```

### 注意
- 当前 Dockerfile 被用户改回了 Alpine 基础镜像，FlClash 是 glibc 编译的，预计仍会遇到 "not found" (exit 127)
- Stage 1 也只复制了单个二进制，未复制 /usr/share/FlClash 整个目录
- 缺少 libgl1/libegl1/GSK_RENDERER=cairo 等之前修复的内容

## 全面修复 Dockerfile（一次性补全所有缺失）

**改动**：

1. **Stage 2 从 Alpine 切回 Ubuntu**：`jlesage/baseimage-gui:alpine-3.23-v4.11.3` → `jlesage/baseimage-gui:ubuntu-24.04-v4`
   - FlClash 是 glibc 编译的 GTK/Flutter 应用，Alpine musl libc 不兼容（exit 127）
   - Ubuntu 变体同样提供 noVNC/VNC GUI，原生 glibc 兼容

2. **Stage 1 复制整个 FlClash 目录**：`cp /usr/share/FlClash/FlClash /dist/FlClash` → `cp -r /usr/share/FlClash /dist/FlClash`
   - FlClash 目录包含二进制 + 资源文件，只复制单个二进制会缺资源

3. **Stage 2 添加所有运行时依赖**：
   - `libgl1`、`libegl1`：Flutter 启动时 dlopen libEGL.so.1，缺则 SIGABRT
   - `libatk-adaptor`：GTK at-spi 辅助功能桥接，缺则断言失败
   - `dbus`、`network-manager`：connectivity_plus 需要 NM 注册 D-Bus 服务
   - `socat`：端口转发
   - `fonts-noto-cjk`、`fonts-wqy-zenhei`：中文字体

4. **添加 D-Bus 宽松配置**：`/etc/dbus-1/system.conf`，允许 ANONYMOUS 连接

5. **添加 NM supervisor 服务**：`/etc/services.d/networkmanager/run`
   - 等待 system_bus_socket 就绪后启动 NM
   - 解决 connectivity_plus CPU 空转问题

6. **添加环境变量**：`GSK_RENDERER=cairo`（VNC 无 GPU，走软件渲染）

7. **compose.yaml 更新**：
   - 启用 `privileged: true`、`cap_add: NET_ADMIN`、`devices: /dev/net/tun`
   - 移除宿主机 dbus socket 挂载（容器内自建）
   - 添加 `build: .` 指令
