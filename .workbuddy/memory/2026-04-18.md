# 2026-04-18 工作日志

## 修复 FlClash Docker 构建失败 (exit code 2)

**报错**: `apt-get install -y /tmp/flclash.deb` 未完成，exit code 2

**根因分析**:
1. FlClash deb 包声明了运行时依赖（libkeybinder-3.0-0、libayatana-appindicator3-1、libgtk-3-0 等），但 Stage 1 基础镜像 `ubuntu:24.04` 未预装这些库
2. 第 11 行安装工具包后立即 `rm -rf /var/lib/apt/lists/*` 清空了 APT 缓存
3. 第 27 行虽然重新 `apt-get update`，但 `apt-get install -y /tmp/flclash.deb` 尝试从仓库拉取 deb 的依赖时失败
4. `/dist` 目录未显式创建，可能导致 `cp` 命令失败

**修复内容**:
- 在 Stage 1 第一个 RUN 中预装 FlClash 所有运行时依赖
- 安装 deb 后再执行 `rm -rf /var/lib/apt/lists/*`
- 显式创建 `/dist` 目录 (`mkdir -p /dist`)
- 修正注释"AppImage"为"deb"
