# 2026-04-18 工作日志

## 修复 FlClash Docker 构建失败 (exit code 2)

**报错**: `apt-get install -y /tmp/flclash.deb` 未完成，exit code 2

**根因分析**:
- FlClash deb 包的 Depends 字段声明的是 `-dev` 开发包（`libayatana-appindicator3-dev`、`libkeybinder-3.0-dev`），不是运行时库（`-1`/`-0` 后缀）
- 之前安装的运行时库包名与 deb 声明不匹配，apt 无法满足依赖约束
- 另外：二进制实际路径是 `/usr/share/FlClash/FlClash`（非 `/usr/bin/flclash`），postinst 创建符号链接

**修复内容**:
- 将 `libkeybinder-3.0-0` → `libkeybinder-3.0-dev`
- 将 `libayatana-appindicator3-1` → `libayatana-appindicator3-dev`
- 移除不必要的多余依赖（libgtk-3-0、libblkid1、liblzma5、libsecret-1-0，-dev 包会自动拉取）
- 修正二进制路径：`/usr/bin/flclash` → `/usr/share/FlClash/FlClash`
- 修正 Stage 2 COPY 路径：`/dist/flclash` → `/dist/FlClash`
- 直接复制实际二进制而非符号链接（符号链接跨 stage 不生效）

## 修复容器运行 FlClash 报 "not found" (exit code 127)

**报错**: `/startapp.sh: exec: line 3: FlClash: not found`，exit code 127

**根因分析**:
- FlClash 是基于 Flutter/GTK 的应用，链接了 glibc（`/lib64/ld-linux-x86-64.so.2`）
- Alpine 使用 musl libc（`/lib/ld-musl-x86-64.so.1`），glibc 二进制无法在 musl 环境执行
- `exec FlClash` 时内核找不到 glibc 动态链接器，返回 "not found"（不是文件不存在，是解释器不存在）

**修复内容**:
- Stage 2 基础镜像从 `jlesage/baseimage-gui:alpine-3.23-v4.11.3` 改为 `jlesage/baseimage-gui:ubuntu-24.04-v4.11.3`
- 运行时依赖从 apk（`add-pkg`）改为 apt-get（`libkeybinder-3.0-0`、`libayatana-appindicator3-1`、`libgtk-3-0`）
- 字体包从 Alpine 包名改为 Ubuntu 包名（`fonts-noto-cjk`、`fonts-wqy-zenhei`）
- 复制完整 `/usr/share/FlClash` 目录（含资源文件），不只是单个二进制
- 在运行层创建符号链接 `/usr/local/bin/FlClash` → `/usr/share/FlClash/FlClash`

## 修复 FlClash 运行时 SIGABRT（缺 libGL + locale）

**报错**:
- `Couldn't open libGL.so.1 or libOpenGL.so.0`
- `Gtk-WARNING: Locale not supported by C library`
- `gdk_gl_context_make_current: assertion failed` → SIGABRT

**根因**:
1. 未安装 OpenGL 运行时库（libgl1），FlClash/Flutter 启动找不到 libGL.so.1
2. 未安装 locales 包并生成 zh_CN.UTF-8，GTK 无法识别 locale
3. VNC 环境无 GPU 硬件加速，Flutter 默认 GL 渲染创建上下文失败后崩溃

**修复**:
- 安装 `libgl1` 提供 libGL.so.1
- 安装 `locales`，`locale-gen zh_CN.UTF-8`
- 设置 `GSK_RENDERER=cairo` 强制 Flutter/GTK 走软件渲染

## 修复 FlClash 运行时 SIGABRT（缺 libEGL.so.1）

**报错**: `Couldn't open libEGL.so.1: libEGL.so.1: cannot open shared object file` → SIGABRT，容器反复重启

**根因**:
- 上一轮只装了 `libgl1`（提供 libGL.so.1），但 Flutter 启动时会先 dlopen libEGL.so.1
- 即便设了 `GSK_RENDERER=cairo`，Flutter 在 GSK 渲染器初始化之前就尝试加载 libEGL.so.1，找不到直接崩溃
- `libegl1` 包提供 libEGL.so.1，必须显式安装
- 另有 `atk_socket_embed: assertion 'plug_id != NULL' failed` 警告，可装 `libatk-adaptor` 消除

**修复**:
- 添加 `libegl1` 包（提供 libEGL.so.1）
- 添加 `libatk-adaptor` 包（ATK 辅助功能桥接）

## 修复 FlClash 容器 CPU 占用过高（connectivity_plus D-Bus 死循环）

**报错**: 容器不再 SIGABRT 崩溃，但 CPU 从 20% 飙到 90%
- `SocketException: Connection failed, address = /var/run/dbus/system_bus_socket` — 无限重试
- `type 'Null' is not a subtype of type 'FutureOr<Directory>'` — AppPath 初始化也受影响
- `libayatana-appindicator-CRITICAL: app_indicator_set_menu: assertion 'IS_APP_INDICATOR (self)' failed`

**根因**:
- FlClash 使用 `connectivity_plus` 插件检测网络连接状态
- 该插件在 Linux 上通过 D-Bus 连接 NetworkManager
- 容器内虽有 dbus session bus（jlesage/baseimage-gui 提供），但没有 **system bus** socket
- `connectivity_plus` 连接 `/var/run/dbus/system_bus_socket` 失败后进入无限重试循环，吃满 CPU

**修复**:
- 安装 `dbus` 包（提供 dbus-daemon 和 dbus-uuidgen）
- 安装 `network-manager` 包（提供 NM D-Bus 服务，否则 connectivity_plus 连上 dbus 也找不到 NM 仍会重试）
- 创建 `/etc/dbus-1/system.conf`（宽松权限配置，允许 ANONYMOUS 连接）
- 在 startapp.sh 中启动 `dbus-daemon --system` + `NetworkManager`
- **重要**：不要挂载宿主机的 `/run/dbus/system_bus_socket`，否则容器内自建的 dbus-daemon 会与挂载的冲突。群晖 DSM 上 NetworkManager 不运行，挂载宿主机 socket 也无用
- 容器内 NM 无需 NET_ADMIN 也可启动，会在 D-Bus 上注册服务并报告状态

## 修复 curl 下载 GitHub release 失败 (exit code 77)

**报错**: `curl -fL -o /tmp/flclash.deb "$URL"` 返回 exit code 77

**根因分析**:
- curl exit code 77 = SSL 证书验证失败（CURLE_SSL_CACERT_BADFILE）
- `ubuntu:24.04` 基础镜像默认未安装 `ca-certificates` 包
- 没有 CA 证书链，curl 无法验证 GitHub 的 HTTPS 证书，请求直接被拒绝

**修复内容**:
- 在 Stage 1 的 apt-get install 中添加 `ca-certificates` 包
