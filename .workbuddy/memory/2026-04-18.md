# 2026-04-18 工作日志

## 修复 FlClash Docker 构建失败 (exit code 2)

**报错**: `apt-get install -y /tmp/flclash.deb` 未完成，exit code 2

**根因分析**:
- FlClash deb 包的 Depends 字段声明的是 `-dev` 开发包（`libayatana-appindicator3-dev`、`libkeybinder-3.0-dev`），不是运行时库（`-1`/`-0` 后缀）
- 之前安装的运行时库包名与 deb 声明不匹配，apt 无法满足依赖约束
- 另外：二进制实际路径是 `/usr/share/FlClash/FlClash`（非 `/usr/bin/flclash`），postinst 创建符号链接

**修复内容**:
- 将 `libkeybinder-3.0-0` → `libkeybinder-3.0-dev`
- 将 `libayatana-appindicator3-1` → `libayatana-appindicator3-dev`
- 移除不必要的多余依赖（libgtk-3-0、libblkid1、liblzma5、libsecret-1-0，-dev 包会自动拉取）
- 修正二进制路径：`/usr/bin/flclash` → `/usr/share/FlClash/FlClash`
- 修正 Stage 2 COPY 路径：`/dist/flclash` → `/dist/FlClash`
- 直接复制实际二进制而非符号链接（符号链接跨 stage 不生效）

## 修复容器运行 FlClash 报 "not found" (exit code 127)

**报错**: `/startapp.sh: exec: line 3: FlClash: not found`，exit code 127

**根因分析**:
- FlClash 是基于 Flutter/GTK 的应用，链接了 glibc（`/lib64/ld-linux-x86-64.so.2`）
- Alpine 使用 musl libc（`/lib/ld-musl-x86-64.so.1`），glibc 二进制无法在 musl 环境执行
- `exec FlClash` 时内核找不到 glibc 动态链接器，返回 "not found"（不是文件不存在，是解释器不存在）

**修复内容**:
- Stage 2 基础镜像从 `jlesage/baseimage-gui:alpine-3.23-v4.11.3` 改为 `jlesage/baseimage-gui:ubuntu-24.04-v4.11.3`
- 运行时依赖从 apk（`add-pkg`）改为 apt-get（`libkeybinder-3.0-0`、`libayatana-appindicator3-1`、`libgtk-3-0`）
- 字体包从 Alpine 包名改为 Ubuntu 包名（`fonts-noto-cjk`、`fonts-wqy-zenhei`）
- 复制完整 `/usr/share/FlClash` 目录（含资源文件），不只是单个二进制
- 在运行层创建符号链接 `/usr/local/bin/FlClash` → `/usr/share/FlClash/FlClash`

## 修复 curl 下载 GitHub release 失败 (exit code 77)

**报错**: `curl -fL -o /tmp/flclash.deb "$URL"` 返回 exit code 77

**根因分析**:
- curl exit code 77 = SSL 证书验证失败（CURLE_SSL_CACERT_BADFILE）
- `ubuntu:24.04` 基础镜像默认未安装 `ca-certificates` 包
- 没有 CA 证书链，curl 无法验证 GitHub 的 HTTPS 证书，请求直接被拒绝

**修复内容**:
- 在 Stage 1 的 apt-get install 中添加 `ca-certificates` 包
